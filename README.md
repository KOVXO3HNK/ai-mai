# Генератор описаний для Handmade

Это веб-приложение помогает создавать красивые и продающие описания для handmade-изделий на основе фотографии.

## Развертывание на Render

Чтобы приложение работало корректно и обходило возможные блокировки, мы развернем его как два отдельных сервиса на Render: **бэкенд (Web Service)** и **фронтенд (Static Site)**.

---

### Шаг 1: Развертывание Бэкенда (Web Service)

Бэкенд будет служить безопасным посредником между вашим приложением и Google Gemini API.

1.  **Войдите в ваш [Dashboard на Render](https://dashboard.render.com/).**
2.  Нажмите кнопку **New +** и выберите **Web Service**.
3.  Подключите ваш репозиторий GitHub.
4.  **Заполните настройки:**
    *   **Name**: `ai-mai-backend` (или любое другое уникальное имя).
    *   **Region**: `Frankfurt (EU Central)` (рекомендуется).
    *   **Branch**: `main`.
    *   **Runtime**: `Node`.
    *   **Build Command**: `npm install`.
    *   **Start Command**: `node server.js`.
5.  Выберите **Free** план.
6.  Нажмите **Advanced**, чтобы добавить переменные окружения.
7.  Нажмите **Add Environment Variable**:
    *   **Key**: `API_KEY`
    *   **Value**: *Вставьте сюда ваш ключ от Google Gemini API.*
8.  Нажмите **Create Web Service**.

Render начнет сборку. Дождитесь, пока статус не изменится на **"Live"**. После этого скопируйте URL вашего бэкенда (он будет выглядеть как `https://ai-mai-backend.onrender.com`). Он понадобится на следующем шаге.

---

### Шаг 2: Развертывание Фронтенда (Static Site)

Фронтенд — это видимая часть вашего сайта, с которой взаимодействует пользователь.

1.  **Вернитесь в [Dashboard на Render](https://dashboard.render.com/).**
2.  Нажмите кнопку **New +** и выберите **Static Site**.
3.  Подключите тот же самый репозиторий GitHub.
4.  **Заполните настройки:**
    *   **Name**: `ai-mai` (или любое другое уникальное имя для вашего сайта).
    *   **Branch**: `main`.
    *   **Build Command**: `npm install && npm run build`.
    *   **Publish Directory**: `dist`.
5.  Нажмите **Advanced**, чтобы добавить переменные окружения.
6.  Нажмите **Add Environment Variable**:
    *   **Key**: `VITE_BACKEND_URL`
    *   **Value**: *Вставьте сюда URL вашего бэкенда, который вы скопировали на Шаге 1.*
7.  Нажмите **Create Static Site**.

Дождитесь завершения развертывания. После этого ваше приложение будет полностью рабочим и доступным по ссылке вашего статического сайта!
